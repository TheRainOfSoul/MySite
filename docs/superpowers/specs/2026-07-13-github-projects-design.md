# Автоматическая секция «Проекты» из GitHub (вкл. приватные)

**Дата:** 2026-07-13
**Статус:** утверждается

## Цель

Секция **Проекты** портфолио должна сама наполняться репозиториями пользователя
`TheRainOfSoul` с GitHub — **публичными и приватными** — с коротким описанием
каждого. Сейчас там три захардкоженные карточки-заглушки; они удаляются.

## Ключевые решения (утверждены пользователем)

| Вопрос | Решение |
|--------|---------|
| Источник описания | Авто + необязательные ручные переопределения |
| Заглушки | Полностью заменить на репозитории с GitHub |
| Языки (RU/EN/HY) | Текст описания — как есть; переводятся только UI-лейблы |
| Приватные репо | Показывать **все** приватные (кроме списка `EXCLUDE`) |
| Хостинг | Netlify (serverless-функция как прокси) |

## Ограничение безопасности (определяет всю архитектуру)

Приватные репозитории требуют GitHub-токен. Сайт статический — весь клиентский
JS открыт публично. **Токен нельзя размещать в браузере.** Поэтому токен живёт
только в серверной функции (переменные окружения Netlify) и в браузер не
попадает — клиент получает лишь готовый безопасный JSON.

> **Важно для пользователя:** при показе всех приватных репо их имена и описания
> становятся публичными. Список `EXCLUDE` на сервере позволяет исключить
> отдельные репозитории — исключённые вообще не покидают сервер.

## Архитектура

Четыре части:

### 1. Serverless-функция `netlify/functions/projects.js`

Единственное место с токеном. Формат — классический `exports.handler`.
Рантайм Node 18+ (глобальный `fetch`, без зависимостей).

Логика:
1. Запросить у GitHub все репозитории владельца токена:
   `GET https://api.github.com/user/repos?visibility=all&affiliation=owner&per_page=100&sort=updated`
   с заголовками `Authorization: Bearer <TOKEN>` и `Accept: application/vnd.github+json`.
2. Отфильтровать: убрать форки (при `excludeForks`), архивные и репозитории из `EXCLUDE`.
3. Для репо без `description` — подтянуть первую осмысленную строку README
   (`GET /repos/{owner}/{repo}/readme`, декодировать base64 как UTF-8, пропустить
   заголовки/бейджи/картинки, обрезать ~140 символов). Токен даёт доступ и к
   приватным README. Ошибки README не критичны (best-effort, `Promise.allSettled`).
4. Нормализовать и вернуть JSON-массив объектов:
   `{ name, title, description, language, topics, stars, updated, htmlUrl, homepage, isPrivate, ogImage }`.
   - `htmlUrl` для приватных = `null` (посторонние получат 404).
   - `ogImage` для приватных = `null` (превью недоступно публично); для публичных —
     `https://opengraph.githubassets.com/1/{owner}/{name}`.
5. Заголовок `Cache-Control: public, s-maxage=3600, stale-while-revalidate=86400`
   — CDN Netlify кэширует ответ ~1 час (меньше вызовов функции и запросов к GitHub;
   лимит с токеном 5000/час).
6. На ошибку GitHub — вернуть `502` с телом `{ error }` (клиент это обрабатывает).

**Конфиг функции (env Netlify):**
- `GITHUB_TOKEN` — секрет, PAT с доступом на чтение нужных репозиториев.
- `GITHUB_USERNAME` — `TheRainOfSoul` (для формирования `ogImage`/README-путей).
- `EXCLUDE_REPOS` — необязательный список имён через запятую; вырезаются на сервере.

**Маршрут:** `netlify.toml` добавляет redirect `/api/projects` →
`/.netlify/functions/projects` и задаёт `NODE_VERSION=18`.

### 2. Клиент `github-projects.js` (новый файл)

Отвечает за загрузку и отрисовку. Сверху — **косметический** конфиг (не секрет,
правится свободно):

```js
const PROJECTS_CONFIG = {
  endpoint: "/api/projects",     // прокси Netlify
  cacheMinutes: 60,              // кэш ответа в localStorage
  fallbackUsername: "TheRainOfSoul", // запасной путь при локальной разработке
  overrides: {
    // "repo-name": { title, description, live, tags, order, hide }
  }
};
```

Поток:
1. Если в `localStorage` свежий кэш (< `cacheMinutes`) — сразу отрисовать из него.
2. Запросить `endpoint`. Успех → отрисовать, обновить кэш.
3. Ошибка → запасной путь: публичный `https://api.github.com/users/{fallbackUsername}/repos`
   (только публичные репо, для локальной разработки без функции).
4. Если и запасной путь упал → показать устаревший кэш (если есть), иначе аккуратное
   сообщение об ошибке со ссылкой на профиль GitHub.
5. Перед отрисовкой применить `overrides` (переопределить описание/заголовок/Live,
   спрятать `hide`, отсортировать по `order`, затем по дате обновления).

### 3. Отрисовка карточек и модалки

Карточка (переиспользует существующие классы `.project-card`, `.glass-card`, `.tilt-card`):
- **Картинка:** `ogImage` (публичные) или градиент-плейсхолдер с именем+языком
  (приватные / если превью не загрузилось — обработчик `onerror`).
- **Заголовок:** `overrides.title` или имя репо, приведённое к читаемому виду
  (`arm-tv` → `Arm Tv`).
- **Описание:** результат цепочки приоритетов
  (override → GitHub description → README → нейтральная строка `«<Язык> проект»`).
- **Теги:** язык + `topics`.
- **Мета:** ⭐ звёзды, «Обновлён: <дата>», бейдж **🔒 Private** для приватных.
  Слова-лейблы берутся из словаря переводов.

Модалка (переиспользует существующий `#project-modal`):
- Заголовок, полное описание, мета.
- Кнопка **GitHub** — только если есть `htmlUrl` (т.е. репо публичный).
- Кнопка **Live Demo** — только если есть `homepage` или `overrides.live`.

### 4. Правки существующих файлов

- **`index.html`:** удалить три статические `.project-card` и их модальные данные;
  оставить контейнер `#projects-grid` (с блоком скелетонов и скрытым блоком ошибки);
  подключить `<script src="github-projects.js">`.
- **`script.js`:** удалить `modal_projects` из всех трёх языков (`ru/en/hy`);
  добавить лейблы-переводы (`proj_stars`, `proj_updated`, `proj_private`,
  `proj_loading`, `proj_error`, `proj_live`, `proj_no_desc` и т.п.);
  вынести эффекты карточек (3D-tilt, свечение под курсором, `.fade-in`-появление,
  hover-курсор) в вызываемую функцию `enhanceProjectCards(nodeList)` и экспортировать
  её (например, `window.enhanceProjectCards`), чтобы `github-projects.js` применял
  эффекты к динамически добавленным карточкам. Существующая обработка модалки
  (`closeModal`, backdrop) переиспользуется; открытие модалки для проектов теперь
  инициирует `github-projects.js` данными репозитория.

## Состояния

- **Загрузка:** карточки-скелетоны в `#projects-grid`.
- **Успех:** сетка карточек.
- **Ошибка:** сообщение «Не удалось загрузить проекты» + ссылка на профиль GitHub;
  сайт продолжает работать.

## Не входит в объём (YAGNI)

- Пагинация (>100 репо не ожидается).
- Языковые переводы самих описаний репозиториев.
- Админ-интерфейс — конфиг правится в файлах.
- Кэш/картинки для локального открытия `file://` (приватные видны только через Netlify).
